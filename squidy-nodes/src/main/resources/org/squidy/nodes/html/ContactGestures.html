Short Description

The ContactGestures node recognizes "Click"-gestures in DataPosition2D's (pre-requirement: identification of contacts with the PositionIdentifier or the Kalman node). For example, this is needed to translate touch contacts from a tabletop to MouseClicks. A Click is recognized when a DataPosition2D is identified without moving for no longer than specified in the parameter "Timeout".
Furthermore, the DataPosition2D must not have moved more than specified in the Parameter "Maximum distance". I.e. this corresponds to a brief tapping with a finger on a tabletop. When a Click was recognized, a DataButton is published to all outbound pipes. For example, a Mouse I/O filter can use these DataButton data to emulate mouse clicks.

Node Type: Filter

Pre-requirement: Identification of contacts with the PositionIdentifier or the Kalman node

Receives on Input Pin: DataPosition2D (e.g. finger contact)

Publish on Output Pin: DataButton (when a click was recognized)
