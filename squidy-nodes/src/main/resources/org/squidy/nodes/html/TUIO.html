Short Description

The TUIO node is a bridge between Squidy and any hardware device or software that uses the TUIO network protocol to transmit position (DataPosition2D) and token (DataDigital) data. The TUIO node can be used both as sender and receiver of TUIO messages depending on the connected node pin (input, output or both).

Node Type: Input and Output

Receives on Input Pin: DataPosition2D (for publishing them on network via TUIO)

Publish on Output Pin: DataPosition2D (e.g. finger contacts), DataDigital (e.g. true when a Token is detected and false when a Token is lifted)

 

Detailed Description

Data Input:

Listens for TUIO Messages with address /tuio/2Dobj, /tuio/2Dcur, /tuio/_sxyXYmaPP or /tuio/_sxyXYma

/tuio/2Dobj represents a Token

/tuio/2Dcur represents a Finger Contact

/tuio/_sxyXYmaPP and /tuio/_sxyXYma are custom addresses for handling finger rotation. These are not yet implemented fully. More information to the TUIO protocol can be found at http://www.tuio.org/?specification

Data Output:

DataPosition2D's are transformed into TUIO bundles according to their attributes. For example, if a DataPosition2D has the attribute TUIO_ORIGIN_ADDRESS with a value of /tuio/2Dcur, the DataPosition2D is transformed into a TUIO 2Dcur message. The TUIO messages are sent via OSC to the port specified through the parameters Remote Address and Remote Port.

When a /tuio/2Dobj message (a token) arrives, a DataDigital is published to all pipes connected at the outgoing side. This is to indicate that a token was set down. The DataDigital has its flag set to true. An additional boolean attribute, TUIO_TOKEN, is attached in order to identify the DataDigital as a token. When the token was lifted, a DataDigital with flag false is sent. Publishing of the DataDigitals happens in the methods publishTokenDown() and publishTokenLifted().
