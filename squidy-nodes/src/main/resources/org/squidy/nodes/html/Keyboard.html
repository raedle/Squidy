Short Description

The Keyboard node is an Input and Output node. It receives as Input Node key events such as a key press or release on the Input Window (starts automatically). These key events are published as DataKey and also as DataDigital on the Output Pin. The Keyboard node can also be used as an Output node since it emulates key presses on the OS layer when it receives DataKey or DataDigital on the Input Pin. An example for such a keyboard emulation and the corresponding DataKey objects can be seen here...

Code Sample for Key-Stroke "STRG+S":

DataKey key1 = new DataKey(className.getClass(),"Control Key", KeyEvent.VK_CONTROL, true);
publish(key1);
DataKey key2 = new DataKey(className.getClass(),"Control Key", KeyEvent.VK_S, true);
publish(key2);
DataKey key3 = new DataKey(className.getClass(),"Control Key", KeyEvent.VK_S, false);
publish(key3);
DataKey key4 = new DataKey(className.getClass(), "Control Key", KeyEvent.VK_CONTROL, false);
publish(key4);

See http://java.sun.com/j2se/1.5.0/docs/api/java/awt/event/KeyEvent.html for a full list of KeyEvent type codes.

Sometimes it is usefull to give Java a small time between consecutive events to process them correctly. This can be done with a Thread.sleep(t)...
// short break for processing the key event
try {
    Thread.sleep(50);
} catch (InterruptedException e) {
    e.printStackTrace();
}

Node Type: Input and Output

Receives on Input Pin: DataKey, DataDigital

Publish on Output Pin: DataKey, DataDigital
